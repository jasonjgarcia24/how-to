1. Create Remote Repository on GitHub

  • Got to github.com
  • In the 'Repositories panel, select New.


  • Fill out the repo name and description.
  • Check Add a README file and if needed Add .gitignore.
    • A '.gitignore' file allows you to populate filenames and file extensions that you do not wish to be included in your commits to the repository.
    • A common use for this file is to prevent config files that have api keys or passwords stored in them from making their way into your repository. -'.gitignore' files come prepopulated with certain files and extension names specific to your programming language, but can be customized to fit your needs.
  • Apply the settings shown in the below image for a standard public repo and click Create repository.


  • Now you should have a blank repo.


2. Clone Repository Locally with SSH

  • Copy your GitHub address for SSH cloning.


  • In Bash type git clone <address>(#git-clone)
  • In this case, `<address>` is 'git@github.com:jasonjgarcia24/fintech-challenges.git')
  • If the note "The authenticity of host 'github.com'... can't be established.... Are you sure you want to continue connecting (yes/no)?", type `yes`.
  • Now you have your local repository and can apply changes as needed locally only.


3. Commit and Push Repository Changes to GitHub

  • In Bash type cd fintech-challenges to enter into the 'fintech-challenges' repository.
  • Apply changes to the repo as necessary.
  • In Bash type git add -A to stage the changes to the Git repository.
    • This is telling the Git application to add any changes to the staging area, and the -A means I want to add all the files in this repository.


  • In Bash type git commit -m "<message>" to commit changes to your remote git. Now you're ready to push the changes.
    • In this case, <message> is 'First commit of challenge 01.')
    • Use -m <message> to type in a commit message. This message is a descriptive message to document a description of the changes.
  • In Bash type git push origin main to push changes to the 'main' GitHub branch.


  • You can now go to your remote repository on github.com, hit refresh, and view the updated repository with the changes applied.


4. Commit to Alternative Branch on GitHub

  • In Bash type git checkout -b <new branch name>.
    • The -b means that you will create a new branch that doesn't already exist.
    • In this case, <new branch name> will be '01-challenge'.
    • Notice also that the branch name in Bash will change from '(main)' to '(01-challenge)'.
  • Apply changes to the repo as necessary.
  • In Bash type git add -A to stage the changes to the Git repository.
    • This is telling the Git application to add any changes to the staging area, and the -A means I want to add all the files in this repository.
  • In Bash type git commit -m "<message>" to commit changes to your remote git. Now you're ready to push the changes.
    • In this case, <message> is 'First commit of challenge 01.')
    • Use -m <message> to type in a commit message. This message is a descriptive message to document a description of the changes.
  • In Bash type git push origin 01-challenge to push changes to the '01-challenge' GitHub branch.


  • You can now go to github.com and hit refresh. You'll notice the 'main' repository has not been updated but that there is now an additional branch.


  • In the dropdown menu next to the number of branches, switch to your new branch (in this case '1-challenge').
  • In the '01-challenge' branch page, click on Compare & pull request.


  • Now you've opened a new "pull request" page.
    • Notice the comment for this branch is the same as what was entered in the Bash window with the git commit -m "<message>" command.


  • Click Create pull request.
  • Since this repo is all yours, you can just merge this pull request right into 'main' by clicking Merge pull request and then Confirm merge.


  • You should now see some form of the message 'Pull request successfully merged and closed'. You can now leave the branch or delete it, it's your preference.
  • Navigate back to the repo's homepage and you can see your alternative branches merged file(s) have been added to the 'main' branch.


5. Checkout Specific Commit with GitBash

  • Go to the homepage of your GitHub repo.
  • Select the commits link to open the full commit history (boxed in yellow below).


  • If previous commits have been pushed to GitHub, you will now see them organized by date.
    • At the far right of each commit you will see, from left to right, a clipboard icon (for copying the commit's hash), the first seven digits of the commit's hash, and an open and close set of angle brackets (for viewing the state of the repository at that commit).
  • For this practice we will be pulling the 'Initial commit' at the bottom of the screen. To do this, click on the clipboard icon of the specific commit that you plan to access. This will copy the full hash to your clipboard.


  • In bash type git checkout <commit hash> to checkout the specific commit.
    • You will then see the branch change from (<current branch>) to ((<hash#...>)). In this case, <current branch> is 'main' and <hash#...> is the first seven digits of the hash, which is 'e106071...'.
  • You now have successfully checked out a previous commit.


Common Git Commands

  • git clone: Clones a remote git repo (e.g. GitHub repo) to the local working directory.
  • git add: Adds changed local repo files and/or folders, as specified, to the queue of tracked files ready to be committed. This is often described as "staging changes".
  • git commit: Finalizes and adds the staged local repo, following a git add ... command, as a single aggregated repo containing all changes. This repo is now ready to be pushed to the remote git repo.
  • git status: Queries and displays status on the current state of the checked out branch on your local repo relative to the same branch on the remote repo.
  • git push: Uploads committed local repo changes to the remote git repo (e.g. GitHub). The remote git repo's folders and files are therefore update to match the local repo's.
  • git pull: Downloads changed files from the remote git repo to the local repo. The local git repo's folders and files are therefore update to match the remote repo's.
  • git checkout: Converts local repo to a specific branch or commit version in the remote repo. All folders and files on the local repo will match the specified branch or commit version.
